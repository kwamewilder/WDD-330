Notes

The Further Functions reading helped me learned many new things about functions in JavaScript.

I did not know functions had a length property.

The call and apply methods are great and I can see they would be great to use in web applications. I personally like the call method better than the apply but they are both great tools.

Cache objects that can store key/value information will be a great property to use in larger scale web apps.

When I work on a new web app, I try to group my functions near the top to make the flow of the program easier to follow. Using the Immediately Invoked Function Expressions (IIFE) could change how I arrange my functions.

An IIFE is a single use function. I would only use this for functions that are required only once.

Functions that define and rewrite themselves could be useful when initializing an app. We could contain all needed startup code in this function and redefine it so nothing that was needed before can affect new functions/functionalities.

The video explianing event loops was great. It helped visualize how JavaScript runs.

Generators are great in storing information after the function has been completed. We could use this to remember important information like points in a game.

Keeping functions as pure as possible seems very important in many aspects but might not be possible in large scale apps.

The Ajax reading helped me learn more about APIs and how they work.

I will have to look into setting up a local development server to prepare for real world server-side development.

I have worked with JSON files and feel very comfortable with them. I will need to learn more about XML files.

The redirect method seems useful if we opened a new tab but it seems odd to redirect since the user would move to a new site page and all other code would be redundant since it would not be used after the page has been redirected. This may be why there is no support for it yet.

I will need to learn about blob objects. I do not know much about them.

I am most familiar with the JSON method when it comes to receiving responses from fetch API.

I am familiar with receiving information to an API but not sending to a database using Fetch API using the POST method.

The FormData interface is new to me and I will need to learn more about it. From what I learned it can help alot when dealing with file uploads.
