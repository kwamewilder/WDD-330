The Transforms and Transitions reading helped me learned many new things about manipulating elements on the page.

I have not dealt with translations before. This will be useful to create banners and general visual elements in a page.

Transforms do not cause a reflow. I will need to remember this when wanting to scale items.

I have not had to use rotation before but I can see this being a helpful tool to adding visuals to our pages.

Transform and transform-origin require prefixing.

Prefixing seems like it could be a bit challenging but doable.

Transition requires the -webkit- prefixing.

There is a list of properties that can be animated, here.

2 seconds/200 milliseconds is considered optimum time for a transition.

IE9 and older does not understand transitions.

The most common timing functions include ease, linear, ease-in, ease-out, and ease-in-out.

Transition-delay may be something to think about using when we don't want to visuals immediately after hovering and want to give the user some time to look over elements on the page.

Short hand transition declarations will save lines of code.

SUing the all keyword in transitions will transition all properties at the same rate, speed and delay.

Animation requires the -webkit- prefix.

Keyframes are snapshots that define a strting or end point of any smooth transition.

The animation name does not have quotes around it in either the property value or the @keyframes at-rule. As the specifications state the name is an identifier and not a string.

Animation-play-state will be helpful to ensure animations don't start from the beginning after pausing.

In the Canvas, SVG, and Drag and Drop reading, I learned many things about images.

Canvas API is not supported by older browser versions.

Canvas drawings are dreated in JavaScript. Because of this, we need to add the height and width in the HTML canvas tag. If not set, the default is 150 height x 300 width

Canvas coordinate system starts at the top left corner down to the bottom right.

strokeStyle draws a border. fillStyle fills path color. lineWidth is used to set the border width.

createLinearGradient x0 and y0 represent starting location. x1 and y1 represent the end location.

Complex shapes require us to layout, stroke and fill the path.

beginPath() resets the default path to begin drawing a new shape.

arc(): arc(x, y, radius, startAngle, endAngle, anticlockwise).

closePath() closes the path.

toDataURL method creates a URL with the image in it.

Reseting the width or height of a canvas will clear it.

To draw text, we need to set the fillStyle and textAlign methods and finally set the text using the fillText method. We can also add a font using the font property of the context object. To separate text to different lines, insert it at different y values.
